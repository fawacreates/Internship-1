# Month 3: High-Value Projects + Portfolio (Weeks 9-12)

## Overview
Build 4 capstone projects that demonstrate end-to-end aerospace software engineering capabilities.

## Project Selection

Choose ONE of the following strategies:

### Strategy A: Deep Dive (2 projects)
- Project 1 (Software) OR Project 2 (Software)
- Project 3 (Hardware) OR Project 4 (Hardware)
- Achievable, high polish

### Strategy B: Full Stack (All 4 projects)
- Complete software AND hardware track
- Massive portfolio impact
- Requires exceptional time management

### Strategy C: Specialization (3 projects)
- Track A: Projects 1, 2, 4 (software + telemetry)
- Track B: Project 2 + 3 + 4 (autonomy + hardware)
- Mixed: Project 1 + 2 + 3 (breadth)

## Projects

1. **[Project 1: Satellite Pipeline](./project-1-satellite-pipeline/)** - Software
   - Target: Pixxel, SatSure
   - Tech: Python, PyTorch, Google Earth Engine
   - Timeline: 3 weeks

2. **[Project 2: Drone Simulator](./project-2-drone-simulator/)** - Software
   - Target: ideaForge, Asteria, Sarla
   - Tech: ROS2, Gazebo, PX4, Python
   - Timeline: 3 weeks

3. **[Project 3: Quadcopter](./project-3-quadcopter/)** - Hardware (Optional)
   - Target: ideaForge, Asteria, Sarla
   - Tech: ArduPilot, PID tuning, telemetry
   - Timeline: 2-3 weeks + field testing
   - Budget: ₹15k-25k

4. **[Project 4: Sensor Payload](./project-4-sensor-payload/)** - Hardware
   - Target: Bellatrix, Ignis, SatSure
   - Tech: ESP32, LoRa, KiCad PCB design
   - Timeline: 2 weeks + assembly
   - Budget: ₹3k-8k

## Portfolio Checklist

By Week 12, ensure each project has:

### GitHub
- [ ] Clear README with problem statement
- [ ] Architecture diagrams
- [ ] Installation and usage instructions
- [ ] Quantified results (accuracy %, test coverage, etc.)
- [ ] Link to demo video

### Documentation
- [ ] Learning journey writeup
- [ ] Technical decisions explained
- [ ] Challenges and solutions
- [ ] Future improvements

### Video
- [ ] 2-5 minute demo
- [ ] Shows working output
- [ ] Explains key features
- [ ] Professional audio/video quality (phone OK)

### Code Quality
- [ ] Clean, commented code
- [ ] Requirements.txt or setup.py
- [ ] Example notebooks or scripts
- [ ] Test cases where applicable
